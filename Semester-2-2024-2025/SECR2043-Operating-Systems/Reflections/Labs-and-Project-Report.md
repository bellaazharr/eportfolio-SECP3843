# Reflection — Operating Systems (Labs & Project Report)

## Objective
Build practical OS knowledge through labs, and apply it in a project (CandyCrushSaga report).

## What I Did
Completed individual labs progressively building OS fundamentals, then contributed to a project report analyzed through an OS lens (CandyCrushSaga).

## What Was Difficult
Connecting abstract OS concepts (scheduling, memory management) to something as seemingly simple as a mobile game required thinking about the application from a systems perspective rather than a user perspective — recognizing what's actually happening underneath a smooth-feeling game.

## How This Connects to Later Coursework
This reframed how I think about "performance" in later courses like High Performance Data Processing — recognizing that performance issues often trace back to OS-level resource constraints (memory, scheduling), not just inefficient code.

## What I'd Do Differently
I'd document the specific OS concepts we mapped onto CandyCrushSaga more explicitly in the report itself, since some of that mapping happened in discussion but didn't make it fully into the written analysis.
