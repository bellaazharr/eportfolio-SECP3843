# Reflection — Data Mining (Supervised & Unsupervised Learning Projects)

## Objective
Apply both supervised and unsupervised learning approaches to real datasets, understanding when each is appropriate.

## What I Did
Completed two projects: one applying supervised learning techniques, the other unsupervised learning, on separate datasets.

## What Was Difficult
The hardest part wasn't implementing either technique — it was correctly framing which problem actually called for labels (supervised) versus pattern discovery without them (unsupervised) before choosing an approach. It's easy to default to supervised learning because it's more commonly taught first, even when the actual business question doesn't have labeled outcomes available.

## How This Connects to Later Coursework
This directly informs how I think about feature design in later projects — for example, deciding whether weather data in my SECP3843 flight price project should be treated as a labeled feature or explored unsupervised first to understand its actual relationship to price before assuming a direct correlation.

## What I'd Do Differently
I'd spend more time on exploratory data analysis before committing to either project's modelling approach, since a few modelling choices in hindsight were made earlier than the data actually justified.
