# SECP3106 — Assignments

## Group Project: E-ducator Timetable System (Client: IKM Johor Bahru)
**Tech:** Flutter, Supabase
**My role:** Module 4 — Upload Time Schedule
**Status:** ✅ Submitted

Built a timetable system for a real client (IKM Johor Bahru) using Flutter for the frontend and Supabase as the backend. My responsibilities included:
- Implementing Module 4 (upload time schedule feature)
- Setting up Flutter from scratch on Windows
- Managing GitHub branches across the team
- Debugging schema mismatches between the app and Supabase

**Design note:** A recurring challenge was schema drift — the Supabase schema and what the Flutter app expected didn't always stay in sync as features were added in parallel by different team members. Debugging this taught me to treat the backend schema as a contract that needs to be communicated, not assumed.
