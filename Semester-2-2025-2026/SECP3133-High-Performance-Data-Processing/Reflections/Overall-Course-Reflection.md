# Overall Course Reflection — SECP3133

## What I Gained
This course reframed "performance" for me — it's not only about writing faster code, it's about choosing the right method before you start. The Mudah.my crawling project made that concrete: switching from BeautifulSoup/Selenium to the site's internal API wasn't an optimization, it was recognizing that the original approach was the wrong tool from the start.

## Suggested Improvement
More explicit framing earlier in the course around when to favor an API over scraping, or Dask over plain Pandas, based on dataset characteristics, would have shortened the trial-and-error I went through on the crawling project.

## Personal Note
The IEEE-format academic writing assignment was unexpectedly valuable alongside the technical work — being precise and rigorous in writing about NoSQL trade-offs turned out to reinforce the same kind of precision needed when actually working with the data itself.
